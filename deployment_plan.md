# Plan: xkx100 вҶ’ Browser Single-Player Game

## Context

The existing xkx100 codebase is a FluffOS LPMUD вҖ” a multiplayer text game running LPC scripts on a custom driver. The goal is to convert it into a **browser-based single-player wuxia RPG** with:
- **Frontend**: React SPA on GitHub Pages (static, free hosting)
- **Backend**: Node.js API + WebSocket on the existing VPS
- **Database**: PostgreSQL on VPS (player save/resume)
- **Auth**: Guest accounts (system-generated name + UUID token), name customizable later
- **UI**: Text + panels layout with ASCII box-drawing, Traditional Chinese throughout
- **Content rollout**: Start with Yangzhou (`/d/city/`) only; expand in patches

The FluffOS driver is **not used** in the new system вҖ” LPC source files are reference material only, mined for room/NPC/skill data via migration scripts.

## Database Schema

```sql
-- sessions: one row per browser/device
CREATE TABLE sessions (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  token        TEXT NOT NULL UNIQUE,       -- stored in localStorage
  player_id    BIGINT,                     -- NULL until player created
  created_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  last_seen_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- players: identity record
CREATE TABLE players (
  id           BIGSERIAL PRIMARY KEY,
  display_name TEXT NOT NULL,              -- customizable
  guest_name   TEXT NOT NULL,              -- system-generated, immutable
  created_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- player_state: all mutable game state
CREATE TABLE player_state (
  player_id    BIGINT PRIMARY KEY REFERENCES players(id) ON DELETE CASCADE,
  current_room TEXT NOT NULL DEFAULT '/d/city/beimen',
  -- vitals
  qi INT, max_qi INT, jing INT, max_jing INT,
  neili INT, max_neili INT, eff_qi INT, eff_jing INT,
  -- base attributes
  str INT, con INT, dex INT, int_stat INT, per INT, kar INT, sta INT, spi INT,
  -- progression
  combat_exp BIGINT DEFAULT 0,
  age INT DEFAULT 14,
  shen INT DEFAULT 0,
  potential INT DEFAULT 0,
  learned_points INT DEFAULT 0,
  money BIGINT DEFAULT 0,
  -- affiliation
  family_name TEXT,
  family_generation INT,
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- player_skills: one row per skill
CREATE TABLE player_skills (
  player_id BIGINT REFERENCES players(id) ON DELETE CASCADE,
  skill_id  TEXT NOT NULL,
  level     INT NOT NULL DEFAULT 0,
  mapped_to TEXT,
  PRIMARY KEY (player_id, skill_id)
);

-- player_inventory
CREATE TABLE player_inventory (
  id        BIGSERIAL PRIMARY KEY,
  player_id BIGINT REFERENCES players(id) ON DELETE CASCADE,
  item_id   TEXT NOT NULL,
  quantity  INT NOT NULL DEFAULT 1,
  slot      TEXT,          -- "weapon" | "armor" | "accessory" | NULL
  extra_data JSONB
);

-- player_quests
CREATE TABLE player_quests (
  player_id    BIGINT REFERENCES players(id) ON DELETE CASCADE,
  quest_id     TEXT NOT NULL,
  status       TEXT NOT NULL DEFAULT 'active',  -- active | completed | failed
  assigned_npc TEXT,
  started_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  completed_at TIMESTAMPTZ,
  PRIMARY KEY (player_id, quest_id)
);
```

## WebSocket Message Protocol

All messages are JSON with a `type` field.

### Client вҶ’ Server

```typescript
type ClientAction =
  | { type: "MOVE";        payload: { direction: string } }
  | { type: "ATTACK";      payload: { targetId: string } }
  | { type: "FLEE" }
  | { type: "TALK";        payload: { npcId: string; topic?: string } }
  | { type: "GET_ITEM";    payload: { itemId: string } }
  | { type: "DROP_ITEM";   payload: { itemId: string } }
  | { type: "USE_ITEM";    payload: { itemId: string } }
  | { type: "EQUIP_ITEM";  payload: { itemId: string } }
  | { type: "LOOK";        payload?: { target?: string } }
  | { type: "TRAIN_SKILL"; payload: { skillId: string; npcId: string } }
  | { type: "REST" }
  | { type: "PING" }
```

### Server вҶ’ Client

```typescript
type ServerEvent =
  | { type: "ROOM_ENTER";      payload: RoomPayload }
  | { type: "LOG";             payload: { messages: LogEntry[] } }
  | { type: "COMBAT_START";    payload: { enemyId: string; enemyName: string } }
  | { type: "COMBAT_ROUND";    payload: CombatRoundPayload }
  | { type: "COMBAT_END";      payload: { result: "win"|"lose"|"flee"; xpGained?: number } }
  | { type: "STAT_UPDATE";     payload: Partial<PlayerStatsPayload> }
  | { type: "INVENTORY_UPDATE";payload: { items: InventoryItem[] } }
  | { type: "DIALOG";          payload: { npcName: string; text: string } }
  | { type: "QUEST_ASSIGNED";  payload: { questId: string; description: string } }
  | { type: "QUEST_COMPLETE";  payload: { questId: string; reward: string } }
  | { type: "SKILL_UPDATE";    payload: { skillId: string; level: number } }
  | { type: "ERROR";           payload: { code: string; message: string } }
  | { type: "PONG" }
  | { type: "TICK";            payload: { tick: number } }
```

Key payload shapes:
- **`RoomPayload`**: `{ roomId, short, long, exits: Record<dir,roomId>, npcs: NpcSummary[], items: ItemSummary[], coords? }`
- **`CombatRoundPayload`**: `{ attackerName, defenderName, skillUsed?, result: "hit"|"dodge"|"parry"|"riposte", damage?, damageType?, message, playerQi, playerJing, enemyQiRatio }`
- **`LogEntry`**: `{ timestamp, category: "move"|"combat"|"system"|"npc"|"item"|"quest", text }`

## LPC вҶ’ JSON Migration Strategy

Migration scripts in `backend/tools/` are run once against the LPC source tree at `/root/projects/xkx/`.

### parse-rooms.ts
- Reads every `*.c` in `/d/city/` (excluding `npc/`, `obj/` subdirs)
- Regex targets: `set("short", "...")`, `set("long", @LONG...LONG)` (heredoc), `set("exits", ([...]))`, `set("objects", ([...]))`, `set("coor/x", N)`, `set("coor/y", N)`, `set("no_fight", 1)`
- `__DIR__` in exits resolved to `/d/city/`. Cross-domain exits kept as absolute path, sealed at runtime if domain not loaded.
- Output: `data/rooms/city/<filename>.json`

### parse-npcs.ts
- Reads every `*.c` in `/d/city/npc/`
- Extracts: `set_name`, `set("long")`, `set("gender")`, `set("age")`, `set("attitude")`, `set("combat_exp")`, stat `set()` calls, `set_skill()`, `set("chat_msg")`, `set("inquiry")`, `carry_object()`, `->wield()`/`->wear()` chains
- `inherit F_DEALER` вҶ’ `"type": "shop"`, `inherit WAITER` вҶ’ `"type": "waiter"`, `inherit F_MASTER` вҶ’ `"type": "trainer"`
- Output: `data/npcs/city/<filename>.json`

### parse-skills.ts
- Reads every `*.c` in `/kungfu/skill/`
- Extracts: `type()`, `martialtype()`, `learn_bonus()`, `success()`, `power_point()`
- Parses `data/e2c_dict.o` (flat key=value format) вҶ’ `data/skills/names.json`
- Output: `data/skills/<skillId>.json`

### parse-items.ts
- Reads `*.c` in `/d/city/obj/` and relevant `/clone/` subdirs
- Detects item type from `inherit` line (SWORD, BLADE, ARMOR, CLOTH, etc.)
- Extracts: `set_name`, `set("value")`, `set("weight")`, `set("long")`, `init_sword(N)` or `init_armor(N)` for stats
